Matrix bugs fixed
🔐
E2EE — bot couldn't decrypt any messages
A temp matrix-js-sdk client was created on every boot with a fresh in-memory crypto store. It uploaded new identity keys to the homeserver, overwriting the bot's real keys. The FE encrypted Megolm room keys to keys the bot could never read.
Fix Deleted the ephemeral client entirely. Cross-signing is a one-time setup at registration.
🧵
Thread root resolved to wrong event
Code walked m.in_reply_to to find the thread root, but that field points at the immediately preceding message — not the root. Caused broken thread tracking for any reply beyond the first.
Fix When rel_type = m.thread is present, use its event_id directly as the root.
🆔
New session created on every message
createSession() was called with event.eventId as the session identifier. Since every Matrix event has a unique ID, every message started a fresh session and conversation context was lost.
Fix Pass checkSessionId (derived from the thread root) — stable across all messages in a thread.
